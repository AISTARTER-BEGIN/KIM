## 1. 앱 개요

### 목적
고객이 원하는 AI 기술 스택과 하드웨어 플랫폼을 선택하면, 예상 개발 기간과 비용을 자동으로 계산해 주는 **온라인 견적 계산기 MVP**. 포트폴리오 사이트에서 실제 영업/협업 문의를 유도하는 CTA(Call to Action) 도구로 활용한다.

### 핵심 가치
- 고객 관점: 복잡한 AI 프로젝트의 비용/기간을 직관적으로 파악
- 개발자 관점: 기술 전문성을 수치로 증명하고, 잠재 클라이언트와의 접점 생성

### 산출물 파일 목록 (예정)
```
my-project/
├── index1.html              # 기존 포트폴리오 (수정: 네비게이션 링크 추가)
├── estimator/
│   ├── index.html           # 견적 계산기 메인 페이지
│   ├── estimator.css        # 전용 스타일 (포트폴리오 CSS 변수 공유)
│   └── estimator.js         # 핵심 비즈니스 로직 (모듈화된 Vanilla JS)
├── data.json                # 기존 파일 (필요 시 옵션 데이터 통합 검토)
└── ROADMAP.md               # 이 파일
```

## 2. 기술 스택 검토

### 결정: 현재 환경(HTML/CSS/Vanilla JS) 완전 유지

| 항목 | 결정 | 이유 |
|------|------|------|
| 프레임워크 | **없음** (Vanilla JS) | 배포 복잡도 제로, 포트폴리오 사이트와 동일한 스택 |
| 빌드 도구 | **없음** | 정적 파일 직접 서빙, GitHub Pages 호환성 유지 |
| 스타일 | **CSS Variables 공유** | 기존 `index1.html`의 `--accent-primary` 등 변수 재사용으로 디자인 일관성 확보 |
| JS 구조 | **IIFE + 모듈 패턴** | `<script type="module">` 또는 즉시실행함수로 전역 오염 방지 |
| 상태 관리 | **일반 객체 리터럴** | 단순한 MVP에 Redux/상태라이브러리 불필요 |
| 데이터 | **JS 상수 객체** | 외부 API 불필요, `PRICING_CONFIG` 상수로 옵션/가중치 관리 |

### 모듈화 전략
```
estimator.js 내부 구조:
  ├── CONFIG 객체       — 모든 옵션 데이터 및 가중치 상수 정의
  ├── calculateEstimate()  — 순수 함수: 입력값 → 결과 반환 (테스트 용이)
  ├── renderResult()    — DOM 업데이트 전담 함수
  └── init()           — 이벤트 리스너 등록, 앱 부트스트랩
```

## 3. 선택 옵션 설계 (초안)

### 3-1. AI 기술 카테고리 (복수 선택 가능)
| 옵션 키 | 레이블 | 기간 가중치 | 비용 가중치 |
|---------|--------|------------|------------|
| `cv_detection` | Computer Vision — 객체 감지 | +3주 | +₩2,500,000 |
| `cv_classification` | Computer Vision — 분류/인식 | +2주 | +₩1,800,000 |
| `sensor_integration` | 센서 데이터 통합 (IMU/환경 센서 등) | +2주 | +₩1,200,000 |
| `edge_inference` | 엣지 AI 추론 최적화 (TFLite/ONNX) | +3주 | +₩2,000,000 |
| `data_pipeline` | 데이터 수집 및 전처리 파이프라인 | +2주 | +₩1,500,000 |
| `dashboard` | 실시간 모니터링 대시보드 | +2주 | +₩1,000,000 |

### 3-2. 하드웨어 플랫폼 (단일 선택)
| 옵션 키 | 레이블 | 기간 가중치 | 비용 가중치 |
|---------|--------|------------|------------|
| `rpi` | Raspberry Pi 5 / CM4 | +1주 | +₩500,000 |
| `jetson` | NVIDIA Jetson (Nano/Orin) | +2주 | +₩1,500,000 |
| `x86_linux` | x86 Linux 서버 | 기준값 | 기준값 |
| `cloud` | 클라우드 (AWS/GCP) | +1주 | +₩800,000/월 |

### 3-3. 프로젝트 복잡도 (단일 선택)
| 옵션 키 | 레이블 | 배수 |
|---------|--------|------|
| `simple` | Simple — 단일 기능, 데모 수준 | × 1.0 |
| `standard` | Standard — 실사용 가능한 MVP | × 1.5 |
| `complex` | Complex — 다중 기능, 안정성 요구 | × 2.2 |

### 3-4. 계산 공식
```
기본값: 기간 4주, 비용 ₩3,000,000

총 기간(주) = (기본 + Σ기술 가중치 + 플랫폼 가중치) × 복잡도 배수
총 비용(원) = (기본 + Σ기술 가중치 + 플랫폼 가중치) × 복잡도 배수
```
> 결과는 **범위(min~max)** 로 표시: min = 계산값 × 0.9, max = 계산값 × 1.2

## 4. 개발 단계 로드맵

### Step 1: UI 설계 — 입력 폼 및 결과 화면
**목표**: 사용자가 직관적으로 옵션을 선택하고 결과를 확인할 수 있는 HTML/CSS 레이아웃 완성

#### 작업 항목
- [ ] `estimator/index.html` 기본 구조 작성 (DOCTYPE, head, meta)
- [ ] 포트폴리오 CSS 변수(`--accent-primary`, `--bg-primary` 등) 재사용 선언
- [ ] **입력 섹션** 구현
  - AI 기술 선택: 체크박스 카드 그리드 (아이콘 + 레이블 + 가격 힌트)
  - 플랫폼 선택: 라디오 버튼 카드
  - 복잡도 선택: 3단계 슬라이더 또는 라디오 버튼
- [ ] **결과 섹션** 구현
  - 예상 기간 범위 표시 (예: `8 ~ 11주`)
  - 예상 비용 범위 표시 (예: `₩12,000,000 ~ ₩16,000,000`)
  - 선택 항목 요약 테이블
  - 문의하기 CTA 버튼 (이메일 링크 or 포트폴리오 contact 섹션 링크)
- [ ] 모바일 반응형 레이아웃 (CSS Grid/Flexbox)
- [ ] 선택 시 카드 하이라이트 애니메이션 (`--transition-base` 재사용)

#### 완료 기준
- 브라우저에서 `estimator/index.html`을 열면 모든 입력 UI가 정상 렌더링됨
- 모바일(375px) ~ 데스크탑(1440px) 해상도에서 레이아웃 깨짐 없음

---

### Step 2: 핵심 비즈니스 로직 구현 — 가중치 연산 JavaScript
**목표**: 선택 옵션을 읽어 기간/비용 범위를 계산하고 실시간으로 DOM에 반영

#### 작업 항목
- [ ] `estimator/estimator.js` 파일 생성
- [ ] `PRICING_CONFIG` 상수 객체 정의 (섹션 3의 가중치 데이터)
- [ ] `calculateEstimate(selections)` 순수 함수 구현
  - 입력: `{ techs: [], platform: string, complexity: string }`
  - 출력: `{ durationMin, durationMax, costMin, costMax }`
- [ ] 체크박스/라디오 변경 이벤트 리스너 — 선택값 수집 → 계산 → 렌더링 파이프라인
- [ ] `renderResult(estimate)` 함수 — 결과 섹션 DOM 업데이트 + 숫자 카운터 애니메이션 (기존 포트폴리오의 카운터 로직 참고)
- [ ] 아무것도 선택하지 않았을 때 결과 섹션 초기 상태 처리
- [ ] 선택값 유효성 검사 (플랫폼은 필수 선택)

#### 완료 기준
- 옵션 선택 즉시 결과가 애니메이션과 함께 업데이트됨
- `calculateEstimate()` 함수는 DOM에 의존하지 않아 독립적으로 테스트 가능
- 브라우저 콘솔에 에러 없음
